# 自动需求采集 Excel 飞书通知设计

## 目标

在不修改 MediaCrawler 核心爬虫逻辑的前提下，新增一个本机单人使用的“自动需求采集”工作流：

1. 系统按固定周期自动生成采集关键词。
2. 通过现有 MediaCrawler CLI 采集公开内容。
3. 读取 MediaCrawler 已导出的 JSONL、CSV 或 SQLite 数据。
4. 优先从评论区提取用户需求和问题，其次从正文内容提取。
5. 去重后生成普通 Excel 文件。
6. 通过飞书群机器人发送采集摘要和本地 Excel 文件路径。

第一版不要求用户手动填写关键词。系统使用内置“领域词 + 需求词”自动组合关键词，先覆盖通用全领域，后续再在 WebUI 中支持勾选领域偏好。

## 不做的事情

- 不修改 MediaCrawler 核心采集逻辑。
- 不接入飞书多维表格。
- 不创建或更新飞书云文档表格。
- 不上传 Excel 文件到飞书。
- 不采集登录墙、私密社群或个人敏感信息。
- 不记录 Cookie、token、飞书 webhook 地址或个人账号到日志。
- 第一版不做多人权限、局域网访问、云端部署和 AI 自动摘要。

## 使用对象

本机单人使用者，希望定时发现各平台公开内容中的用户需求、痛点、问题、求推荐和消费决策线索，并在飞书收到提醒后打开本地 Excel 查看。

## 核心工作流

```text
定时器触发
→ 自动选择平台、领域词、需求词
→ 生成关键词组合
→ 调用 MediaCrawler CLI
→ 读取本地导出文件
→ 识别评论/正文中的需求内容
→ 去重
→ 生成 Excel
→ 飞书机器人通知摘要和本地文件路径
```

## 自动关键词策略

第一版使用内置词库，不要求用户输入关键词。

### 领域词

初始领域覆盖通用需求场景：

- AI工具
- 装修
- 旅游
- 副业
- 母婴
- 数码
- 美妆
- 教育
- 职场
- 健康
- 宠物
- 家居
- 汽车
- 餐饮
- 本地生活

### 需求词

需求词用于提高采集结果里“用户问题/需求”的比例：

- 求推荐
- 避坑
- 踩雷
- 哪个好
- 值不值
- 怎么选
- 怎么买
- 预算
- 平替
- 后悔
- 有没有必要
- 攻略
- 推荐一下
- 真实体验
- 注意事项

### 组合规则

每次任务从领域词和需求词中随机或轮询组合，例如：

```text
装修 避坑
AI工具 推荐
旅游 攻略
母婴 怎么选
数码 值不值
```

第一版默认全领域轮换。后续 WebUI 可加入领域勾选，但不是第一版必需能力。

## 采集入口

新增工作流只调用现有 MediaCrawler CLI，不直接修改各平台 crawler：

```text
uv run python main.py --platform <platform> --type search --keywords <auto_keyword> ...
```

第一版优先支持 search 模式，因为它最适合自动关键词组合。detail 和 creator 模式保留给原有 WebUI 或手动任务，不纳入自动需求采集 MVP。

## 定时策略

第一版支持以下周期：

- 手动运行一次
- 每 3 小时
- 每 6 小时
- 每 12 小时
- 每天
- 每周

定时配置保存在本地配置文件中，不依赖飞书表格。任务运行时需要避免重入：如果上一轮还在执行，下一轮应跳过并记录日志。

## 需求提取规则

需求提取只做轻量规则，不引入 AI 服务。

优先级：

1. 评论内容
2. 正文标题和正文描述

满足以下任一条件的文本可进入候选：

- 包含需求词或问题词，如“求推荐”“哪个好”“怎么选”“有没有必要”“预算”“避坑”。
- 包含问号或明显提问句式。
- 中文内容长度达到最小阈值，避免无意义短文本。

跳过以下内容：

- 原文内容为空。
- 中文字符过少。
- 纯表情、纯链接、纯账号信息。
- 明显广告、抽奖、无关互动话术。

## 去重策略

使用内容哈希去重：

```text
sha256(platform + source_url + raw_text)
```

同步状态保存在本地：

```text
.sync_state/demand_excel_synced_hashes.json
```

同一条评论或正文重复出现时不重复写入新的 Excel 结果。

## Excel 输出

默认输出 `.xlsx` 文件，适合直接打开和筛选。

输出目录：

```text
output/demand_reports/
```

文件名格式：

```text
YYYY-MM-DD-HHMM-demand-report.xlsx
```

字段：

| 字段 | 说明 |
| --- | --- |
| 需求标题 | 从原文截取生成，便于快速浏览 |
| 需求原文 | 评论或正文原文 |
| 内容类型 | 评论 / 正文 |
| 来源平台 | xhs / dy / bili / wb / zhihu 等 |
| 自动关键词 | 本轮自动生成的关键词 |
| 领域 | 自动关键词中的领域词 |
| 需求词 | 自动关键词中的需求词 |
| 来源链接 | 原文链接 |
| 作者/昵称 | 数据中存在则填写 |
| 发布时间 | 平台内容发布时间 |
| 采集时间 | 本地采集时间 |
| 内容哈希 | 去重证据 |
| 备注 | 空，供人工填写 |

## 飞书通知

第一版只使用飞书群机器人 webhook。

需要配置：

```text
FEISHU_WEBHOOK_URL=<飞书群机器人 webhook>
```

通知内容示例：

```text
MediaCrawler 自动需求采集完成

周期：每 6 小时
平台：小红书、微博、知乎
本轮关键词：装修 避坑 / AI工具 推荐 / 旅游 攻略
新增需求：126 条
评论需求：98 条
正文需求：28 条
跳过重复：34 条
失败任务：0 个

Excel 文件：
E:\codex\GitHub项目\MediaCrawler\output\demand_reports\2026-06-27-1800-demand-report.xlsx
```

通知不上传文件，只发送摘要和本机文件路径。

## WebUI 范围

第一版 WebUI 只需要让本机用户完成以下操作：

- 查看自动采集是否开启。
- 选择周期：3 小时、6 小时、12 小时、每天、每周。
- 手动运行一次。
- 查看最近一次 Excel 输出路径。
- 查看最近运行日志。
- 配置或检查飞书 webhook 是否存在。

第一版不需要做领域勾选。领域勾选作为后续增强。

## 模块边界

### 自动关键词生成器

职责：

- 维护内置领域词和需求词。
- 生成本轮关键词组合。
- 记录关键词、领域、需求词，供 Excel 输出使用。

不负责：

- 平台采集。
- 内容解析。
- 飞书通知。

### 定时任务管理器

职责：

- 管理周期配置。
- 防止任务重入。
- 支持手动运行一次。
- 调用自动采集工作流。

不负责：

- 修改 MediaCrawler 核心爬虫。
- 长期云端调度。

### 采集执行器

职责：

- 使用现有 MediaCrawler CLI 执行 search 采集。
- 指定输出目录。
- 收集导出的 JSONL、CSV 或 SQLite 文件路径。

不负责：

- 直接调用平台 API。
- 修改各平台 crawler。

### 需求整理器

职责：

- 读取导出文件。
- 提取评论和正文中的需求候选。
- 去重。
- 输出统一结构。

不负责：

- AI 分类。
- 飞书发送。

### Excel 生成器

职责：

- 生成 `.xlsx`。
- 写入字段和基础筛选。
- 返回本地文件路径。

不负责：

- 上传文件。
- 云端表格写入。

### 飞书通知器

职责：

- 通过 webhook 发送本轮摘要。
- 不打印 webhook 完整地址。
- 失败时记录脱敏错误。

不负责：

- 多维表格写入。
- 文件上传。

## 错误处理

- MediaCrawler 执行失败：记录失败任务，继续处理其他任务。
- 没有导出文件：本轮 Excel 不生成，飞书通知说明“无新增结果”。
- Excel 生成失败：飞书通知失败原因，日志保留异常类型和脱敏信息。
- 飞书通知失败：本地日志记录失败，不影响 Excel 文件保留。
- 上一轮还在运行：跳过本轮，记录“上一轮未结束”。

## 安全约束

- 只采集公开内容。
- 不在 WebUI 中输入 Cookie。
- 不在日志中输出 Cookie、token、webhook 完整地址。
- 飞书通知只发送摘要和本地路径，不发送原始隐私信息。
- Excel 保留来源链接和原文，方便人工追溯。

## 验收标准

### 功能验收

- 可以手动运行一次自动需求采集。
- 可以配置 3 小时、6 小时、12 小时、每天、每周周期。
- 自动采集不要求用户填写关键词。
- 生成的 Excel 文件存在且可打开。
- Excel 中至少包含评论需求和正文需求两类内容。
- 飞书收到本轮摘要和本地 Excel 路径。
- 重复内容不会重复进入结果。

### 技术验收

- 不修改 MediaCrawler 核心 crawler 目录逻辑。
- 调用 MediaCrawler 仍通过 CLI。
- 飞书通知不依赖多维表格。
- 所有外部飞书请求在测试中必须 mock。
- pytest 全部通过。

### 安全验收

- 日志不包含 Cookie、token、webhook 完整地址。
- `.env` 不提交。
- 截图、凭据、本地状态文件不进入提交。

## 后续增强

- WebUI 支持领域勾选。
- 支持每个平台独立配置采集数量。
- 支持 Excel 中自动标记需求类型。
- 支持将 Excel 文件上传到飞书云盘。
- 支持 AI 摘要和需求聚类。
