## 1. 项目概述

MediaCrawler 是一个多平台公开内容采集 + AI 需求分析工具。采集完成后自动对数据进行 LLM 去重、关键词分类、AI 产品方案生成，并将分析报告推送到 WebUI 和飞书群。

**核心链路：** 采集 → LLM 去重筛选 → 关键词分类 → AI 方案生成 → 报告持久化 → WebSocket 推送前端 + 飞书通知

| 平台 | 标识 | 支持方式 |
| --- | --- | --- |
| 小红书 | xhs | 关键词、指定ID、创作者 |
| 抖音 | dy | 关键词、指定ID、创作者 |
| 快手 | ks | 关键词、指定ID、创作者 |
| Bilibili | bili | 关键词、指定ID、创作者 |
| 微博 | wb | 关键词、指定ID、创作者 |
| 贴吧 | tieba | 关键词、指定ID、创作者 |
| 知乎 | zhihu | 关键词、指定ID、创作者 |

### v2.0 新增功能

- **AI 需求分析报告**：采集完成后自动分析，全新可视化报告页面（KPI 卡片、痛点分布图、产品类型饼图、热度矩阵）
- **LLM 语义分类**：使用大模型对内容进行语义理解分类，新增食品/生活/餐饮等分类
- **LLM 去重筛选**：自动识别相似内容和噪声，过滤无意义数据
- **报告持久化**：分析结果保存到 `data/reports/`，刷新页面不丢失
- **WebSocket 实时推送**：分析完成后自动推送到前端展示
- **飞书通知精简**：仅发送关键摘要（Top 5 痛点 + 1 条 AI 建议）
- **小红书反检测强化**：Cookie 全量注入、CDP 模式注入 stealth.js、延迟随机化、UA 一致性修复

## 2. 环境要求

- Python 3.11+
- uv
- Node.js 16+（部分平台签名或文档构建场景需要）
- Chrome 浏览器（推荐用于 CDP 模式）

### 核心依赖

```powershell
cd E:\codex\GitHub项目\MediaCrawler
uv sync
```

### 可选依赖

以下依赖在需要时才安装，缺失不阻塞启动：

| 依赖 | 用途 | 安装命令 |
| --- | --- | --- |
| sqlalchemy + aiosqlite | SQLite 存储 | `uv add sqlalchemy aiosqlite` |
| sqlalchemy + asyncmy | MySQL 存储 | `uv add sqlalchemy asyncmy` |
| openpyxl | Excel 导出 | `uv add openpyxl` |
| opencv-python | 抖音滑块验证 | `uv add opencv-python` |
| jieba | 中文分词（备选分类器） | `uv add jieba` |

如需标准 Playwright 浏览器模式：

```powershell
uv run playwright install
```

## 3. 快速开始

### 启动 WebUI

推荐直接使用项目根目录的启动脚本：

```powershell
.\启动WebUI.bat
```

也可以手动启动：

```powershell
uv run python -m api.main
```

或：

```powershell
python -m uvicorn api.main:app --host 0.0.0.0 --port 8081
```

启动后在浏览器访问：

```
http://localhost:8081
```

> 注意：`0.0.0.0` 是服务监听地址，不要用 `http://0.0.0.0:8081` 访问。

## 4. 配置说明

### 环境变量

项目读取根目录 `.env` 文件。核心变量：

| 变量 | 说明 | 使用场景 |
| --- | --- | --- |
| `FEISHU_WEBHOOK_URL` | 飞书群机器人 Webhook | 采集完成 / 分析报告通知 |
| `LLM_API_KEY` | AI 模型 API Key | LLM 去重 + 语义分类 + AI 方案生成 |
| `LLM_API_URL` | AI 模型接口地址 | 默认 DeepSeek API |
| `LLM_MODEL` | AI 模型名称 | 默认 `deepseek-v4-flash` |
| `WEBUI_API_KEY` | WebUI 访问密码（可选） | API 认证保护 |
| `COOKIES` | 平台 Cookie 字符串 | Cookie 登录方式 |

### .env 示例

```env
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
LLM_API_KEY=sk-xxx
LLM_API_URL=https://api.deepseek.com/v1/chat/completions
LLM_MODEL=deepseek-v4-flash
```

### AI 模型说明

配置 LLM 后可启用三大能力：

| 能力 | 消耗 | 说明 |
| --- | --- | --- |
| 数据去重 | 低 | 每批 200 条一次调用，识别语义重复和噪声 |
| 语义分类 | 中 | 每批 30 条一次调用，12 个分类精准匹配 |
| AI 方案生成 | 高 | 每个分类一次调用，生成产品方案 |

**不配置 LLM_API_KEY 时**：去重回退为 hash 去重，分类回退为关键词分词匹配，方案回退为内置模板。基础功能不受影响。

### 分类体系

系统内置 12 个需求分类：

| 分类 | 覆盖范围 |
| --- | --- |
| 食品安全 & 成分健康 | 配料、添加剂、营养 |
| 零食饮料 & 口味体验 | 零食测评、新品推荐 |
| 餐饮探店 & 美食体验 | 餐厅推荐、就餐体验 |
| 生活用品 & 家居购物 | 好物推荐、家居收纳 |
| 内容创作 & AI 写作 | 文案、自媒体创作 |
| 自动化 & 效率工具 | 批量处理、流程优化 |
| 数据分析 & 可视化 | 数据报表、监控看板 |
| 电商 & 选品工具 | 选品、跨境、竞品分析 |
| 学习 & 技能提升 | 教程、培训、自学 |
| 社交 & 社区运营 | 涨粉、社群、私域 |
| 开发者工具 & 脚本 | API、插件、开源 |
| 其他需求 & 未分类 | 兜底分类 |

## 5. Dashboard 使用指南

左侧导航：

| 页面 | 功能 |
| --- | --- |
| 总览 | 本机采集控制 + 运行日志 |
| 分析报告 | AI 需求分析报告（采集后自动生成） |
| 需求库 | 数据文件管理、预览、搜索、导出 |

### 总览

- **本机直接采集**：填写参数后一键启动采集，支持 7 个平台 × 3 种模式。
- **运行日志**：实时查看采集和分析日志，支持按级别和来源过滤。

### 分析报告

采集完成后自动推送到此页面，也可在需求库中手动生成。报告内容包括：

- **6 个 KPI 指标卡片**：总记录、需求分类、高热度需求、高价值机会、AI 方案、Webhook 状态
- **AI 总评**：自动生成分析摘要和下一步建议
- **三个可视化图表**：痛点分布柱状图、产品类型饼图、热度矩阵象限图
- **AI 产品机会卡片**：Top 6 痛点方案，含优先级、描述、MVP 功能
- **需求机会明细表**：9 列全维度可筛选表格
- **详情抽屉**：点击"查看详情"查看代表内容摘录、匹配关键词、AI 方案详情

报告数据持久化到 `data/reports/`，刷新不丢失。页面加载时自动获取最新报告。

### 需求库

查看 `data/` 目录下所有导出文件，支持：

- 按平台/格式/关键词过滤和排序
- 预览数据内容（JSONL/CSV/Excel/SQLite）
- 单文件下载
- 多文件批量 ZIP 导出
- 全文本搜索
- **分析报告**按钮：对文件执行完整分析流程

### 配置检查

检查本机环境和各组件状态。

### Webhook / AI 模型

飞书群机器人和 LLM 接口的密钥配置。

## 6. 本机采集流程

1. 打开 `http://localhost:8081`，点击左侧 `总览`。
2. 在 `本机直接采集` 区域选择平台和采集类型。
3. 填写关键词 / 指定 ID / 创作者 ID。
4. 选择登录方式、保存格式、起始页、最大内容数。
5. 按需勾选评论抓取、无头浏览器。
6. 勾选 `我确认仅采集公开内容并遵守平台规则`。
7. 点击 `启动采集`。
8. 采集完成后，**分析报告自动生成并推送**到分析报告页面。
9. 若配置了飞书 Webhook，群内同步收到摘要通知。

如需停止，点击 `停止采集`。

## 7. AI 需求分析报告

### 自动触发

采集完成后，系统自动执行以下流程：

```
记录加载 → LLM 去重筛选 → 语义分类 → AI 方案生成
                                            ↓
                          ┌─────────────────┤
                          ↓                 ↓
                    report_store.save   send_demand_report
                    (JSON 持久化)       (飞书摘要通知)
                          ↓
                  WebSocket 实时推送
                          ↓
                  前端分析报告页面
                    自动渲染展示
```

### 手动生成

1. 进入 `需求库` 页面。
2. 找到目标 JSONL 数据文件。
3. 点击 `分析报告` 按钮。
4. 确认后系统执行完整分析流程。
5. 结果自动显示在分析报告页面。

### 报告持久化

所有分析报告以 JSON 格式保存在 `data/reports/` 目录，保留最近 50 份。API 端点：

| 端点 | 方法 | 说明 |
| --- | --- | --- |
| `/api/data/analyze-report` | POST | 对指定文件执行分析 |
| `/api/data/analysis-reports/latest` | GET | 获取最新报告 |
| `/api/data/analysis-reports` | GET | 获取报告历史列表 |

## 8. 数据存储与导出

采集数据默认保存在 `data/<platform>/jsonl/` 目录，常见格式：JSONL / CSV / SQLite / JSON / Excel。

在 `需求库` 页面可进行：

- **预览**：查看数据内容（最多 100 条）
- **下载**：单文件或 ZIP 批量下载
- **搜索**：全文关键词检索
- **分析**：触发 AI 需求分析

1. 打开飞书群 → 群设置 → 添加自定义机器人。
2. 复制 Webhook URL。
3. 在 Dashboard 的 `Webhook` 页面粘贴 URL 并保存。

## 10. 小红书反检测优化

v2.0 针对小红书平台的反爬检测进行了多项强化：

| 优化项 | 说明 |
| --- | --- |
| Cookie 全量注入 | 登录时注入所有 Cookie（a1、gid、webId 等），不再只注 web_session |
| CDP 模式 stealth.js | CDP 模式下也注入 stealth.min.js + 专用反检测 JS |
| 延迟随机化 | 所有请求间隔从固定值改为随机范围（0.5x ~ 1.5x） |
| UA 一致性 | 浏览器和 HTTP 客户端使用相同的 User-Agent |
| 反检测 JS | 修补 navigator.webdriver、plugins、chrome.runtime、permissions、cdc_ 痕迹 |

### Cookie 登录注意事项

使用 Cookie 登录时，请从浏览器开发者工具复制**完整的 Cookie 字符串**（包含 a1、gid、web_session 等所有字段），不要只复制 web_session。

## 11. 常见问题

### Q1：为什么 `http://0.0.0.0:8081` 打不开？

`0.0.0.0` 是服务监听地址，浏览器应访问 `http://localhost:8081` 或 `http://127.0.0.1:8081`。

### Q2：如何确认 WebUI 服务正常？

访问 `http://localhost:8081/api/health`，正常返回 `{"status":"ok"}`。

### Q3：启动报错 ModuleNotFoundError？

部分依赖为可选依赖，缺失不阻塞启动。详见 [可选依赖](#可选依赖) 表格。

### Q4：采集时弹出二维码怎么办？

首次采集需要扫码登录，用对应平台 App 扫码后在浏览器中保持会话。

### Q5：采集完成但没有看到分析报告？

检查以下步骤：
1. 日志中是否有 "Report saved & pushed" 提示
2. 确认 `LLM_API_KEY` 已配置（无 Key 时分类和方案使用备选模式）
3. 在 `需求库` 中手动点击文件的 "分析报告" 按钮测试
4. 检查 `data/reports/` 目录是否生成了 JSON 文件

### Q6：分析报告中的分类不准确？

配置 `LLM_API_KEY` 后系统优先使用 LLM 语义分类（准确度大幅提升）。未配置时使用关键词分词匹配。

### Q7：端口被占用怎么办？

```powershell
# 查看端口占用
Get-NetTCPConnection -LocalPort 8081 -State Listen

# 换端口启动
python -m uvicorn api.main:app --host 0.0.0.0 --port 8082
```

### Q8：CDP 端口 9222 不可访问？

确认 Chrome 已开启远程调试，访问 `http://localhost:9222/json` 验证。不使用 CDP 时可切换为标准 Playwright 模式。

### Q9：小红书 Cookie 登录后被警告？

参见 [小红书反检测优化](#10-小红书反检测优化)。确保使用完整的 Cookie 字符串（包含 a1、gid、web_session 等）。避免短时间内高频采集。

### Q10：飞书没有收到通知？

1. 确认 `FEISHU_WEBHOOK_URL` 已配置且通过测试连接
2. 检查 Dashboard 日志中是否有 "Analysis report sent" 或 "Summary sent" 提示
3. 飞书自定义机器人的消息频率限制为 20 条/分钟
